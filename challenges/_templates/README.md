# Challenge Templates

These templates are starting points for new SteerLab challenges.

Use the template that matches the archetype you are authoring:

- `broken-system-investigation/`
- `architecture-thought-experiment/`
- `tool-steering-challenge/`

Suggested workflow:

1. Copy the relevant template folder to a new challenge directory.
2. Replace placeholder values in `challenge.yaml`.
3. Add only the artifacts needed to support disciplined reasoning.
4. Run `npm run validate:challenges`.
5. Re-read [`docs/challenge-design-guide.md`](../../docs/challenge-design-guide.md) before opening a PR.
