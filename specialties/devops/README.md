# DevOps Specialty Pack

This pack covers operational systems, deployment workflows, infrastructure reliability, and environment behavior that product engineering teams plausibly encounter.

It is meant to support challenge generation in areas like:

- deployment failures
- rollback gaps
- observability blind spots
- configuration drift
- operational tradeoffs around delivery speed and safety

It should stay grounded in realistic product-team operations rather than generic cloud certification trivia.

When extending this pack:

- prefer incidents and tradeoffs that can be reasoned about from supplied artifacts
- avoid vendor-name swapping that does not change the underlying engineering judgment
- keep the focus on systems behavior, validation, and operability
