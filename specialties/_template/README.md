# Specialty Pack Template

Use this directory as the starting point for a new specialty pack.

The pack should define a realistic engineering domain that can be used across one or more of the canonical archetypes.

Good pack contributions should:

- improve domain realism
- clarify artifact expectations
- surface realistic failure modes or tradeoff axes
- avoid creating a hidden fourth archetype

Start with `pack.yaml`, then use this README to explain the domain boundaries and contribution expectations for the pack.
